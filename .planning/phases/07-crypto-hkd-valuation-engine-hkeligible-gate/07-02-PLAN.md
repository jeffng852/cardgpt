<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| caller → engine (injected rateTable) | The `HkdRateTable` is untrusted input crossing into the engine; a zero/negative/NaN/stale rate could produce a garbage HKD valuation that mis-ranks the crypto segment or crashes |
| card data → valuation | `rewardPrograms.crypto` names the asset used as the rate-table key; a mis-keyed/absent asset must degrade safely, not throw |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-VAL | Tampering | `valuateCrypto` rate handling | mitigate | Validate `rate.hkdPerUnit > 0` (rejects 0, negative, NaN, non-number) and require a resolvable asset + parseable `asOf`; degrade to `hkdEquivalent: null` ("value unavailable") on any bad input rather than trusting the number; unit tests assert zero/negative/NaN/absent all yield null |
| T-07-VAL-b | Denial of Service | `valuateCrypto` on malformed asOf | mitigate | `Date.parse` NaN → treat as stale (finite check), never throw; no unbounded work — pure O(1) lookup + arithmetic |
| T-07-STK | Elevation of Privilege | `minStaking` gate in `matchesRule` | mitigate | Fail-closed: a staking-gated tier does NOT match by default, so an unstaked user is never valued at a tier they haven't unlocked (base tier only); tested against a two-tier synthetic card |
</threat_model>
