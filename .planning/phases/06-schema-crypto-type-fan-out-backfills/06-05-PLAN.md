<objective>
Backfill `cardType: "credit"` onto every card in PRODUCTION Upstash Redis (CRY-01's second half — the live truth). This is CardGPT's schema-push analog: because there is no ORM, `npx tsc --noEmit` and `next build` pass as soon as the types + `cards.json` are updated (06-01), even though production Redis is still serving type-less cards. That is a false "done." This plan makes the live backfill a distinct, explicit, human-gated deliverable and verifies against Redis itself — build/typecheck passing is INSUFFICIENT proof for CRY-01.

The backfill MUST be merge-aware: read the current `cards` object from Redis, set `cardType` on each card that lacks one (preserving every other field and any live admin edit), then write the object back. It MUST NOT use `/api/admin/init-redis` or any path that writes `cards.json` over Redis — that clobbers live admin edits (see the Phase 5 CDN/Redis fight and the redis_backfill rule).

This repo is PUBLIC: the script must not hardcode secrets; read prod env via `vercel env pull` (CLI is linked). Log the Linear issue (THI-XXX) BEFORE running against prod (issue-first is absolute), and reference it in the commit.

Purpose: Make the live product carry the required cardType — the real CRY-01 deliverable.
Output: `scripts/backfill-card-type.mjs` + a verified prod Redis state.
</objective>
